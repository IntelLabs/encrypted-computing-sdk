96 
97  For all keygen variables of the same seed, `copy` instructions are injected and
98  all instructions using the same keygen variable as the `copy` instruction become
99  dependent of said `copy`. This ensures that the variable is generated before it is
100  used. Furthermore, `copy` instructions depend on each other based on the ordering
101  of the keygen variables. This ensures correct ordering of key material generation.
102 
103  Parameters:
104  deps_graph (nx.DiGraph): Completed graph of dependencies among the instructions in the input listing.
105  Will be changed with the injected instructions and dependencies.
106  mem_model (MemoryModel): Completed memory model corresponding to the specified dependency graph and
107  input mem info. Will be changed if new variables are needed during injection.
108  verbose_ostream: Stream where to print verbose output (object with a `write` method).
109  If None, no verbose output occurs.
110  """
111

113  #
114  # copy kg_var_0 -> op kg_var_0
115  # -> op kg_var_0
116  # -> op kg_var_0
117  # ...
118  # -> copy kg_var_1 -> op kg_var_1
119  # -> op kg_var_1
120  # -> op kg_var_1
121  # ...
122  # -> copy kg_var_2 -> ...
123  #
124  # This ordering ensures that kg_var_X is generated the first time its copy
125  # instruction is met and all the other uses can then occur with the generated
126  # value. Then, kg_var_X+1 can also be generated after kg_var_X has been generated
127  # without needing to wait for all kg_var_X uses to occur (only the first copy).

142  # Create temp target variable

150  # Enforce ordering of copies based on ordering of keygen
